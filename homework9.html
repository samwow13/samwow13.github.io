How To upload to the cloud

In this section I'll just basically write out the process of uploading to the cloud. This walkthrough should show that I understand the process, as well as complete the necessary blog requirements.

  1. First, start by creating an account on Azure.
  2. Then, click resource group off to the left. Setup a new resource with the plus sign, name it what you like. I went with FallTerm. Don't forget to refresh!
  3. Time to setup the database. Add a new db, blank, no elastic pool. Select create server, fill in necessary fields. Then make sure to use the basic pricing tier, then pin it to your dashboard!
  4. Next up, create a new web application. Use the existing resource group, then fill in all necessary fields and pin to dashboard.
  5. Do a quick check to see if it is live!
  6. Now, click on your DB and find "set server firewall". Whatever your listed IP is, go ahead and add it so you can securely access the DB.
  7. Next up, open Visual Studio and login to Azure.
  8. Open Sql Server Management Studio and login properly here as well. Example below.
  9. Now, go ahead and run your Up and populate script on the database.
  10. Add a local connection string to the Online Database. For now, set it here and hide it later. Remember, no brackets around the username and password information!
  11. Add the dbcontext name as well as the connection string to your Azure/Appsettings/connectionStrings area.
  12. Rebuild!
  13. Now its time to publish! Select existing, azure services, publish!
  14. Test your web app!

Conclusion

I think Azure can be a great place for hosting websites with databases. Pretty simple to setup once everything works. Once you get the process down, should be a very quick way to deploy websites!
